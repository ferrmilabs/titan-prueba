GisImageApi Reference

Implements access to GIS related Texture (Remote Sensing image).

Access to Texture Image data

Texture data is stored as a Raster image data. Possible sources of material information are remote sensing images.

Texture File Format

               Texture is loaded from three files, the header file, the data file, and the colors file. The format of the files is similar to GRASS format.

·               Header file:

               Is a text (ASCII) file located in cellhd directory, with the format:

 

 

Notes:

proj:

<projection identification code> 

API does not check this code.

zone:

<UTM zone number>

API does not check this code.

north:

<maximum Y coordinate>

 

south:

<minimum Y coordinate>

 

east:

<maximum X coordinate>

 

west:

<minimum Y coordinate>

 

cols:

<number of columns>

 

rows:

<number of rows>

 

e-w resol:

<resolution of a cell in X-direction>

e-w and n-s must be equal

n-s resol:

<resolution of a cell in Y-direction>

 

format:

0

Indicates byte value at the cell

compressed:

<compression flag>

0 – uncompressed; 1 - compressed

 

·               Data file:

               Is a binary file located in cell directory, with the number of rows times number of columns byte values. The data sequence is west to east (left to right), north to south (top to bottom). Byte values are indices to color table stored in Colors file.

·               Colors file:

               Is a text (ASCII) file located in colr directory, with the format:

 

 

Notes:

% <first color id> <last color id>

 

 

0:<r0>:<g0>:<b0>

 

R, G, and B for color 0

1:<r1>:<g1>:<b1>

 

R, G, and B for color 1

2:<r2>:<g2>:<b2>

 

R, G, and B for color 2

3:<r3>:<g3>:<b3>

 

R, G, and B for color 3

4:<r4>:<g4>:<b4>

 

R, G, and B for color 4

…..

 

 

…..

 

 

<last color id>:<r>:<g>:<b>

 

R, G, and B for last color

 

Usage Notes

 

·               Memory management:

               All space is managed by calling functions and must allocate and deallocate memory.

 

·               X and Y extents:

               X and Y maximum and minimum coordinates does not refer to a grid cell center but to the extremes of the cell.

               The difference between X maximum and X minimum will always be equal to the number of columns multiplied by the resolution. The same applies to Y direction and the number of rows.

               Therefore number of rows and columns are given by:

ncols = (ymax-ymin) / res

nrows =(xmax-xmin) / res

 

·               Grid row and column coordinates and XY coordinates:

               The grid cell at row zero and column zero has coordinates Xmin and Ymax;

               The grid cell at row nrows-1 and column ncols-1 has coordinates Xmax and Ymin.

              

·               Required source files:

               GisApi.h : The API include file

               GisApi.C: Implementation of the API functions

               GisAscFile.C; GisAscFile.h: Auxiliary to API, handle ASCII format files

               GisBinFile.C; GisBinFile.h: Auxiliary to API, handle binary format files

               GisRasterHdr.C; GisRasterHdr.h: Auxiliary to API, handle header files

               GisColors.C; GisColors.h: Auxiliary to API, handle categories files
