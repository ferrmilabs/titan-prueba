GIS Interface

Implements access to GIS related data: Digital Elevation Model (DEM), Material (usually geology or land use) map, and Texture (Remote Sensing image). DEM and Material provide data for the numerical simulation and Texture is used for visualization. The GIS data must be in raster format.

The format of GIS data is based on GRASS format, which allows data to be manipulated by GRASS. However, GIS data is not dependent of GRASS, and using the specifications of the data formats, one can create data from any other GIS software or from another source.

GIS data is accessed by the application program through an interface (Application Program Interface – API). The main features of the GIS-API are:

-         Data is loaded from files to memory during initialization, assuming that there are enough systems resources.

-         When data is not required anymore, application program must deallocated by calling API delete functions.

-         Data can be obtained in resolution different than the original data. The nearest neighbor to the grid cell center is used for all cases, except when the resolution is smaller for DEM, in which case a bilinear interpolation is used.

-         Data can be accessed for individual locations, a set of locations, or for a regular grid area.

-         Slope and curvature data are derived automatically from DEM, using an eight neighbor finite difference method.

-         Slope and curvature are calculated in the original resolution.

Files naming conventions:

-         DEM data defines the base name.

-         Material data with the same extents of the DEM is named using the base name added by “_Mat”.

-         Material data with extents that covers the DEM is named “Material”.

-         Texture data with the same extents of the DEM is named using the base name added by “_Img”.

-         Texture data with extents that covers the DEM is named “Image”.

Files location:

-         All header files must reside in <GISDbase>/ <location>/<mapset>/cellhd directory.

-         DEM data file must reside in <GISDbase>/ <location>/<mapset>/fcell directory.

-         Material data file must reside in <GISDbase>/ <location>/<mapset>/cell directory.

-         Texture data file must reside in <GISDbase>/ <location>/<mapset>/cell directory.

-         Material categories file must reside in <GISDbase>/ <location>/<mapset>/cats directory.

-         Texture colors file must reside in <GISDbase>/ <location>/<mapset>/colr directory.

GISAPI Usage Notes

 

·               Memory management:

               All space is managed by calling functions and must allocate and deallocate memory.

 

·               X and Y extents:

               X and Y maximum and minimum coordinates does not refer to a grid cell center but to the extremes of the cell.

               The difference between X maximum and X minimum is equal to the number of columns multiplied by the resolution.

               The same applies to Y direction and the number of rows.

               Therefore number of rows and columns are given by:

                                         ncols = (ymax-ymin) / res

                                         nrows =(xmax-xmin) / res

 

·               Grid row and column coordinates and XY coordinates:

               The grid cell at row zero and column zero has coordinates Xmin and Ymax;

               The grid cell at row nrows-1 and column ncols-1 has coordinates Xmax and Ymin.

 

·               Resolution in X and Y direction must be equal:

               This assumption is required for slope and curvature calculation method.

               When data is loaded, a difference of 10% is allowed.

               The extents in Y direction of the data will be recalculated from the X direction resolution.

              

·               Implementation in C++, using STL.

-    Application must include:

               GisApi.h : The API include file.

-    Application must link with a library built using:

               GisApi.h : The API include file.

               GisApi.C: Implementation of the API functions

               GisAscFile.C; GisAscFile.h: Auxiliary to API, handle ASCII format files

               GisBinFile.C; GisBinFile.h: Auxiliary to API, handle binary format files

               GisRasterHdr.C; GisRasterHdr.h: Auxiliary to API, handle header files

GisCats.C;  GisCats.h: Auxiliary to API, handle Material categories files

GisColors.C; GisColors.h: Auxiliary to API, handle Texture colors files
